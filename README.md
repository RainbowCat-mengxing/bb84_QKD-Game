# 量子密钥局：BB84 交互小游戏（使用CodeX 开发）

一个无需安装依赖、可直接在浏览器中运行的 BB84 量子密钥分发教学游戏。玩家可扮演 Alice、Bob 或窃听者 Eve，自由控制消息编码、制备/测量基、信道噪声、抽样比例、警戒线和拦截等等变量。
若扮演通信方，游戏目标是确保通信的安全，识破可能的窃听者；
若扮演窃听方，游戏目标是想方设法窃取更多的信息，减少被发现的可能性。
此项目作为《量子信息基础》课程的大作业，开发目的是：以更直观的方式，帮助我自己和所有玩家加深对 BB84 量子密钥分发协议的理解。游戏界面内有协议速查板块，供玩家随时参考。
## 运行

下载本仓库的文件后，直接双击 `index.html`或者访问链接： `https://rainbowcat-mengxing.github.io/bb84_QKD-Game/`

## 玩法

- Alice：修改原始密钥位和制备基，查看公开抽样 QBER 后决定中止或继续。
- Bob：逐位选择测量基，筛基后判断是否有 Eve。
- Eve：设置拦截比例和测量基；单击光子切换基，双击切换该位是否拦截。
- 支持 UTF-8、ASCII 和直接比特输入。
- 默认使用随机 Eve、1% 固有噪声、25% 公开抽样和 15% 中止警戒线；所有参数均可修改。
- 游戏消息，数据等保存在浏览器本地存储中。

## 协议模型

本项目没有把明文直接当作 BB84 光子负载。正确流程是：

1. 消息转换为 ASCII/UTF-8 比特，仅用于确定所需密钥长度。
2. Alice 生成随机原始密钥，并以随机 Z/X 基制备量子态。
3. Bob 选择基测量；基不一致时结果等概率随机。
4. Eve 的拦截—测量—重发遵循相同测量规则，因此会扰动量子态。
5. Alice 与 Bob 公开选基并筛除不一致位，再公开一部分结果估计 QBER。
6. 若决定继续，游戏以剩余密钥执行一次一密。实际系统还必须进行纠错、认证与隐私放大；

理想全量拦截—重发攻击在 Alice/Bob 筛后密钥上造成约 25% 的期望 QBER。这个数是攻击特征值，并非任何实现都能直接采用的普适安全阈值；游戏特意允许调整警戒线，以体现噪声、有限抽样和误判之间的权衡。
